## 3. Pipeline de ingesta de datos (ETL)

### 3.1 INEI — Precios de referencia
```
Programación: mensual (cuando INEI publica nuevo índice)

1. Verificar si hay nuevo .xlsx en inei.gob.pe
2. Si hay: descargar → parsear con openpyxl → upsert a tabla `precios_referencia`
3. Si no hay: mantener datos del mes anterior
4. Registrar versión/fecha de la tabla usada (RF-SCO-11)
```

### 3.2 SEACE / OCDS — Metadatos de obra
```
Programación: diaria (o bajo demanda)

1. Consultar API OCDS por rango de fechas o por entidad
2. Extraer: obra (título, monto, ubicación), contratista (RUC, razón social),
   entidad, fechas
3. Obtener coordenadas desde campo de ubicación o geocodificar
4. Upsert a tablas `obras`, `contratistas`, `contratos`
```

### 3.3 INFOBRAS — Partidas del expediente vía Gemini
```
Programación: bajo demanda (cuando se consulta una obra por primera vez)

1. Obtener URL del PDF del expediente desde INFOBRAS
2. Descargar PDF → convertir a base64
3. Enviar a Gemini API con prompt estructurado:
   "Extrae la tabla de partidas del siguiente expediente técnico.
    Devuelve un array JSON con objetos {insumo, unidad, cantidad,
    precio_unitario}. Usa estos nombres exactos."
4. Validar respuesta:
   - ¿Tiene los campos requeridos?
   - ¿Los precios están en rangos esperados? (ej. cemento > 0 y < 200)
   - Si validación OK → insertar en `partidas_obra`
   - Si no → marcar obra como "fallback" y usar RF-SCO-08
5. Almacenar respuesta cruda de Gemini en `log_extraccion` para auditoría
6. Cachear resultado para no reprocesar en consultas subsecuentes
```

### 3.4 JNE — Autoridades
```
Programación: semanal

1. Descargar dataset de autoridades electas/vigentes
   (datosabiertos.gob.pe — JNE)
2. Parsear CSV/XLSX → upsert a tabla `autoridades`
3. Vincular autoridades con municipio y período
```

## 4. Modelo de datos (entidades principales)

```
obras
├── id (UUID, PK)
├── codigo_infobras (único)
├── titulo
├── tipo_obra (enum: edificacion, carretera, agua_saneamiento, etc.)
├── estado (enum: planeado, ejecucion, concluido, paralizado)
├── presupuesto_total (decimal)
├── metrado_total (decimal, m²)
├── ubicacion (geography: Point — PostGIS)
├── departamento, provincia, distrito
├── entidad_ejecutora_id (FK → entidades)
├── contratista_id (FK → contratistas)
├── score_riesgo (int, 0-100)
├── modo_analisis (enum: partidas, fallback_m2)
├── fecha_extraccion
└── fecha_actualizacion

partidas_obra
├── id (UUID, PK)
├── obra_id (FK → obras)
├── insumo (varchar)
├── codigo_inei (varchar, nullable — para matching)
├── unidad (varchar)
├── cantidad (decimal)
├── precio_declarado (decimal)
├── precio_referencia_inei (decimal, nullable)
├── ratio (decimal, nullable)
└── version_tabla_inei (varchar)

precios_referencia
├── id (UUID, PK)
├── codigo_inei (varchar, indexed)
├── insumo (varchar)
├── unidad (varchar)
├── precio (decimal)
├── departamento (varchar, nullable — ajuste regional)
├── mes (int)
├── anio (int)
└── fuente (enum: inei, mvivienda)

contratistas
├── id (UUID, PK)
├── ruc (varchar, unique)
├── razon_social
├── representante_legal
├── estado_sunat
├── score_confiabilidad (int, 0-100)
└── fecha_actualizacion

entidades (municipios)
├── id (UUID, PK)
├── nombre
├── tipo (enum: municipalidad_distrital, municipalidad_provincial,
│           gobierno_regional, etc.)
├── departamento, provincia, distrito
├── ubigeo

autoridades
├── id (UUID, PK)
├── entidad_id (FK → entidades)
├── nombre
├── cargo (enum: alcalde, regidor, gobernador, etc.)
├── partido
├── periodo_inicio, periodo_fin
├── foto_url (nullable)
└── fuente_actualizacion

log_extraccion
├── id (UUID, PK)
├── obra_id (FK → obras)
├── fuente (enum: gemini, inei, seace, sunat, jne)
├── respuesta_cruda (JSONB)
├── exitoso (boolean)
├── intentos (int)
└── created_at (timestamp)

usuarios
├── id (UUID, PK)
├── email (unique)
├── password_hash
├── alias (público)
├── rol (enum: anonimo, registrado, administrador)
└── created_at

comentarios
├── id (UUID, PK)
├── usuario_id (FK → usuarios)
├── recurso_tipo (enum: obra, empresa, municipio, autoridad)
├── recurso_id (UUID — FK polimórfica)
├── contenido (text)
├── padre_id (FK → comentarios, nullable — respuestas anidadas)
├── reportado (boolean)
└── created_at

suscripciones
├── id (UUID, PK)
├── usuario_id (FK → usuarios)
├── recurso_tipo (enum: obra, empresa, municipio, autoridad)
├── recurso_id (UUID)
└── created_at
```

## 5. Stack tecnológico detallado

| Capa | Tecnología | Versión | Justificación |
|---|---|---|---|
| Frontend | Next.js | 14.x (App Router) | SSR para SEO (RNF-12), Leaflet para mapas, shadcn/ui para componentes |
| Backend | FastAPI | 0.110+ | Asíncrono nativo, integración fácil con PostgreSQL async, OpenAPI auto |
| Base de datos | PostgreSQL + PostGIS | 15+ | Datos geoespaciales (obras por ubicación), consultas espaciales nativas |
| Cache | Redis (o en memoria vía FastAPI) | — | Límites de API de fuentes externas (RNF-05) |
| ETL programado | APScheduler | — | Sincronización periódica simple, sin necesidad de Celery para MVP |
| Scraping extras | httpx + BeautifulSoup4 | — | JNE, SUNAT (fuentes web no API) |
| Extracción PDF | Gemini API (google-generativeai) | — | Ver ADR-001 |
| Parseo Excel | openpyxl | — | INEI .xlsx mensual |

## 6. Atributos de calidad

| Atributo | Estrategia |
|---|---|
| **Disponibilidad** | Las fuentes externas se cachean. Si fallan, se sirve el último dato cacheado con fecha visible. |
| **Performance mapa** | Clustering de marcadores + carga por viewport. Scores precomputados, no en cada request. |
| **Seguridad** | Solo datos públicos. Passwords hasheadas. Backoffice inaccesible sin rol admin. Rate limiting en endpoints públicos. |
| **Mantenibilidad** | `DataSource` interface aísla cada fuente externa. Cambiar de Latinfo a scraper directo no afecta el motor de scoring. |
| **Testeabilidad** | Motor de scoring se testea contra `MockDataSource` sin depender de redes ni créditos. |
